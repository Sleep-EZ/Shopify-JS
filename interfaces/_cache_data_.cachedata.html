Interface CacheData

The CacheData is the data structure used for caching fetched Shopify objects. Objects are indexed by their Shopify ID (all types in the Shopify platform have an ID), and another index is managed for quickly looking up items by their type and handle.

NOTE: You should never need to access this object directly. You should always use one of Client's common methods that calls get(), or cache expiration will not work.

NOTE: Shopify types can have colliding handles. For example, a Product with the handle example, and a Page with the same handle, can exist on the same store. So, we must index handles categorized by their type as well.

Object Format:

{
  ids: {
    1: { __type: 'page', id: 1, handle: 'example' }
  },
  handles: { "page": { "example": 1 } },
}
